🍸 Cocktailmixer API

Developer Documentation v2.0 100% AI-SLOP and Lazy made

Base URL: http://localhost:5000
🍹 Cocktails 🧪 Ingredients 🎯 Order 🔒 PIN Auth 📊 Status
GET /api/cocktails

Ruft alle verfügbaren Cocktails ab. Optional filterbar nach alkoholisch/alkoholfrei.

Query Parameters
Parameter Type Description
alkoholisch string "true" für alkoholische, "false" für alkoholfreie Cocktails
Example Request
GET http://localhost:5000/api/cocktails
GET http://localhost:5000/api/cocktails?alkoholisch=true
GET http://localhost:5000/api/cocktails?alkoholisch=false
Example Response
[
  {
    "id": 1,
    "name": "Gin Tonic",
    "liquid_recipe": [
      {
        "pump_id": 8,
        "ingredient_name": "Tonic",
        "amount_ml": 260
      }
    ],
    "manual_ingredients": [
      {
        "ingredient_name": "Limette",
        "instruction": "1 Scheibe hinzufügen"
      }
    ],
    "alkoholisch": true,
    "glass_size_ml": 350
  }
]
GET /api/ingredients

Gibt den aktuellen Status aller Zutaten zurück (Füllstand, ID, ob pumpbar).

Example Response
[
  {
    "ingredient_id": 1,
    "ingredient_name": "Cola",
    "is_liquid": true,
    "current_level": 750,
    "pump_id": 0
  }
]
POST /api/ingredients/refill

Füllt eine spezifische Zutat auf (addiert zur aktuellen Menge).

Request Body REQUIRED 
{
  "ingredient_id": 1,
  "amount": 200
}
Example Response
{
  "success": true,
  "ingredient_id": 1,
  "added_amount": 200,
  "new_level": 950
}
POST /api/ingredients/refill_all

Setzt alle Zutaten auf einen bestimmten Wert (Standard: 2000ml).

Request Body OPTIONAL 
{
  "level": 2000
}
Example Response
{
  "success": true,
  "level_set": 2000,
  "updated_count": 19
}
POST /api/order

Bestellt einen Cocktail und startet den automatischen Mixvorgang.

Request Body REQUIRED 
{
  "cocktail_id": 1
}
Example Response
{
  "status": "mixing",
  "cocktail": "Gin Tonic",
  "alkoholisch": true,
  "volume": "350ml",
  "liquid_ingredients": [...],
  "manual_steps": [
    {
      "ingredient_name": "Limette",
      "instruction": "1 Scheibe hinzufügen"
    }
  ]
}
POST /api/check-pin

Überprüft die eingegebene PIN für alkoholische Getränke.

Request Body REQUIRED 
{
  "pin": "1234"
}
Example Response (Success)
{
  "valid": true,
  "message": "PIN korrekt"
}
Example Response (Error)
{
  "valid": false,
  "message": "PIN falsch"
}
POST /api/change-pin

Ändert die PIN (benötigt die alte PIN zur Verifizierung).

Request Body REQUIRED 
{
  "old_pin": "1234",
  "new_pin": "5678"
}
Example Response
{
  "success": true,
  "message": "PIN erfolgreich geändert"
}
GET /api/status

Gibt den aktuellen Systemstatus zurück (Mixing-Status, Anzahl Cocktails).

Example Response
{
  "is_mixing": false,
  "total_cocktails": 30,
  "alcoholic_cocktails": 22,
  "non_alcoholic_cocktails": 8
}
