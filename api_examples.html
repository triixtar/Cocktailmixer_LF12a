🍹 Cocktails
GET /cocktails

Gibt alle Cocktails zurück, für die aktuell genügend Zutaten vorhanden sind. Enthält sowohl pumpbare Zutaten (liquid_recipe) als auch manuelle Schritte (manual_ingredients). Optional nach Alkohol filterbar.

Query Parameters
ParameterTypeValuesDescription
alkoholischstring"true" / "false"Filtert nach alkoholisch / alkoholfrei. Ohne Parameter: alle Cocktails.
Example Requests
GET /api/cocktails
    GET /api/cocktails?alkoholisch=true
    GET /api/cocktails?alkoholisch=false

🎯 Order
POST /order

Startet den Mixvorgang für einen Cocktail. Die Pumpen laufen im Hintergrund-Thread. Wenn der Cocktail manuelle Zutaten erfordert, werden entsprechende Anweisungen zurückgegeben.

Request Body REQUIRED
{
    "cocktail_id": 1
    }
Example Response
{
    "status": "mixing",
    "cocktail": "Gin Tonic",
    "alkoholisch": true,
    "volume": "350ml",
    "liquid_ingredients": [
        { "pump_id": 12, "ingredient_name": "Gin", "amount_ml": 50 },
        { "pump_id": 8,  "ingredient_name": "Tonic", "amount_ml": 260 }
    ],
    "manual_steps": [
        {
        "ingredient_name": "Limette",
        "instruction": "Schneide 1 Limettenscheibe(n) und gib sie ins Glas"
        }
    ],
    "message": "Cocktail wird gemischt. Bitte folgende Zutaten manuell hinzufügen:",
    "instructions": [
        "Schneide 1 Limettenscheibe(n) und gib sie ins Glas"
    ]
    }
Error Response
{
    "error": "Cocktail nicht verfügbar"
    }
🧪 Ingredients
GET /ingredients

Status aller Zutaten: Füllstand, ob pumpbar, zugehörige Pumpen-ID.

Example Response
[
    {
        "ingredient_id": 1,
        "ingredient_name": "Cola",
        "is_liquid": true,
        "current_level": 750,
        "max_level": 2000,
        "pump_id": 0
    },
    {
        "ingredient_id": 17,
        "ingredient_name": "Limette",
        "is_liquid": false,
        "current_level": 10,
        "max_level": 20,
        "pump_id": null
    }
    ]
POST /ingredients/set

Setzt den Füllstand einer Zutat auf einen exakten Wert. Setzt gleichzeitig maxLevel auf denselben Wert.

Request Body REQUIRED
{
    "ingredient_id": 1,
    "level": 1500
    }
Example Response
{
    "success": true,
    "ingredient_id": 1,
    "new_level": 1500,
    "max_level": 1500,
    "message": "Zutat 1 auf 1500ml gesetzt"
    }
POST /ingredients/refill

Addiert eine Menge zum aktuellen Füllstand einer Zutat. Erhöht maxLevel automatisch, falls der neue Level den bisherigen Maximalwert übersteigt.

Request Body REQUIRED
{
    "ingredient_id": 1,
    "amount": 500
    }
Example Response
{
    "success": true,
    "ingredient_id": 1,
    "added_amount": 500,
    "new_level": 1250,
    "max_level": 2000,
    "message": "Zutat 1: +500ml hinzugefügt"
    }
POST /ingredients/refill_all

Setzt alle Zutaten auf einen bestimmten Füllstand. Nützlich zum schnellen Reset.

Request Body OPTIONAL
{
    "level": 2000
    }
    // default: 2000 wenn weggelassen
Example Response
{
    "success": true,
    "level_set": 2000,
    "updated_count": 19,
    "ingredients_status": [ ... ],
    "message": "Alle 19 Zutaten auf 2000ml gesetzt"
    }
⚙️ Pumps
POST /test-pump/{pump_id}

Lässt eine einzelne Pumpe für 2 Sekunden laufen. Zum manuellen Testen der Hardware. pump_id entspricht ingredientID - 1.

URL Parameter
ParameterTypeDescription
pump_idintegerPumpen-ID (0-basiert, z.B. 0 = Cola, 12 = Gin)
Example Request
POST /api/test-pump/12
Example Response
{
    "success": true,
    "pump_id": 12
    }
POST /pump/{pump_id}/start

Schaltet eine Pumpe dauerhaft ein — ohne Timeout. Gedacht für Button-Hold-Interaktion im Frontend (mousedown → start, mouseup → stop).

URL Parameter
ParameterTypeDescription
pump_idintegerPumpen-ID (0-basiert)
Example Request
POST /api/pump/12/start
Example Response
{
    "success": true,
    "pump_id": 12
    }
POST /pump/{pump_id}/stop

Schaltet eine laufende Pumpe aus. Sollte immer nach /pump/{id}/start aufgerufen werden.

URL Parameter
ParameterTypeDescription
pump_idintegerPumpen-ID (0-basiert)
Example Request
POST /api/pump/12/stop
Example Response
{
    "success": true,
    "pump_id": 12
    }
🔒 PIN Auth
POST /check-pin

Überprüft eine 4-stellige PIN. Zweck über purpose steuerbar: "alcohol" prüft gegen die konfigurierbare Alkohol-PIN, "admin" prüft gegen die feste Admin-PIN.

Request Body REQUIRED
{
    "pin": "1234",
    "purpose": "alcohol"   // "alcohol" (default) | "admin"
    }
Example Response (valid)
{
    "valid": true,
    "message": "PIN korrekt"
    }
Example Response (invalid)
{
    "valid": false,
    "message": "PIN falsch"
    }
POST /change-pin

Ändert die Alkohol-PIN. Erfordert die aktuelle PIN zur Verifizierung. Die Admin-PIN ist nicht änderbar über die API.

Request Body REQUIRED
{
    "old_pin": "1234",
    "new_pin": "5678"
    }
Example Response
{
    "success": true,
    "message": "PIN erfolgreich geändert"
    }
Error Response (wrong old PIN)
{
    "success": false,
    "message": "Alte PIN falsch"
    }  // HTTP 401
